### Updating scs-0219

sponsor: @mbuechse

- issue [standards#1097](https://github.com/SovereignCloudStack/standards/issues/1097), PR [standards#1166](https://github.com/SovereignCloudStack/standards/pull/1166)
- add recommendation of Gateway API ("Gateway API is the successor to the Ingress API")
- neither Gateway API nor Ingress API are currently covered by any testcase, so not only does it not influence certification (because it is only recommendation), compliance with it will not even show up anywhere
- precedence: we added recommended image metadata field `os_purpose`
  - however, this was arguably a minor change
- in fact, Gateway API does not replace Ingress API; and Ingress API will continue to exist! Gateway API a new, parallel paradigm, so to speak
  - so we shouldn't replace Ingress wholesale
  - OTOH, Ingress was never a real Kubernetes standard, as opposed to Gateway
- we replace the mention of Ingress only in a list of examples
  - this could be construed as expressing a value judgment
  - we might change this PR: continue to mention Ingress?
  - or indeed bump version?
- bump version to make clear that we indeed value Gateway higher than Ingress
  - include version history (as usual) with some explanation
