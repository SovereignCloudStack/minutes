### KaaS testing

sponsor: @mbuechse

- do we test k8s or do we test managed k8s?
  - we currently test properties of clusters
  - BUT we do expect each active k8s branch to be available
  - and we do expect test clusters to be created on demand via our plugin system
  - therefore, it could be argued that we test managed k8s (at least in a rudimentary way)
- potential problems
  - keep test subjects current with active k8s branches
  - add plugins for new lifecycle management systems (such as Gardener)
- do we deem the automated tests critical for certification
  - ... or can they be implemented after the fact with some deadline
  - this approach has proved to work quite badly in the past
  - we do want to look for critical CVEs though
- there is potential for simplification
  - we as SCS are not required to automatically test everything (or anything)
  - it would, for instance, be possible to only test the latest k8s branch available at the CSP
